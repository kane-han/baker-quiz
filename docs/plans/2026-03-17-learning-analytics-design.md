# 학습 분석 & 취약 복습 시스템 설계

## 개요
사용자가 자신의 약점을 파악하고 취약 문제를 효과적으로 복습할 수 있는 시스템.

## 1. 데이터 저장 (LocalStorage)

**새 키: `baker-quiz-log`** — 모든 문제 풀이 기록

```json
[{"id":"b-3-42", "picked":1, "correct":true, "date":"2026-03-17"}, ...]
```

- `id` 포맷: `{cat 첫글자}-{시험인덱스}-{문제인덱스}`
- 챕터 모드: 문제 텍스트 앞 20자 해시로 원본 문제와 매핑
- 기존 `baker-quiz-history`(시험 단위 기록)는 그대로 유지

## 2. 홈 화면 요약

기존 통계 영역 아래에 한 줄 인사이트 추가:
> ⚠️ **식품위생학**이 가장 약해요 (정답률 45%) → 복습하기

풀이 기록 없으면 미표시.

## 3. 학습 분석 화면

홈에 "📊 학습 분석" 카드로 진입.

- **섹션 A: 챕터별 정답률** — 5개 챕터 CSS 막대그래프
- **섹션 B: 오답 패턴** — 자주 틀리는 문제 TOP 10 리스트 (틀린 횟수, 마지막 틀린 날짜, 고른 오답 보기)

## 4. 취약 문제 자동 복습

가중치 기반 20문제 출제.

- **공식:** `weight = 틀린횟수 - (맞힌횟수 × 0.5)`
- weight > 0인 문제만 대상
- weight 높은 순 정렬 후 상위에서 랜덤 추출
- 대상 20개 미만이면 있는 만큼만 출제
- 기존 퀴즈 UI 재사용 (챕터 모드 패턴)

## 5. 제약사항

- 외부 라이브러리 없음 (Vanilla JS/CSS)
- 차트는 CSS 막대그래프
- 기존 코드 최소 수정 (새 함수 추가 위주)
